export & imports

Allows to split the code in multiple files - content (modules). Inside a JS file we can import content of another file.


FROM person.js file:
const person = {
name: 'X'
}
export default person //this will always be the DEFAULT EXPORT

FROM utility.js file:
export const clean = () => {...} //NAMED EXPORT
export const baseData = 10;

IN app.js:
import person from './person.js'
import prs from './person.js' //because the import is default - we can use whatever we want as a name
import { baseData } from './utility.js' //we need to give it the exact name, because it is not exported as default
import { clean } from './utility.js'
COULD BE DONE ALSO LIKE: import { baseData, clean } from './utility.js'
OR ALIAS: import { baseData as Base } from './utility.js'
OR ALL: import * as bundled from './utility.js'
